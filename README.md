# Gesture MIDI Instrument

A two-track collaborative music system. A musician plays their instrument while a dancer's body movements and expressions shape the background music generated by MRT2 — no music knowledge required for the dancer.

---

## How It Works

```
MUSICIAN TRACK                      DANCER TRACK
──────────────────────────────      ──────────────────────────────
MIDI keyboard → Reaper → audio      Hands → MIDI notes → MRT2
Head nods → tempo clock             Face + body → MRT2 parameters
Autocorrelation → BPM refinement    Background music generation
```

Both tracks run simultaneously on one machine. Audio from Reaper and MRT2 mix to the same speakers.

---

## Requirements

- macOS (tested on macOS 14+)
- Python 3.9+
- Two cameras: MacBook built-in + iPhone via Continuity Camera (or USB webcam)
- MRT2 Jam (App Store, bundle ID: `com.google.mrt2.jam`)
- Reaper (for musician audio output)

## Track 1 — Musician (`musician.py`)

The musician plays their MIDI keyboard normally. This script handles tempo sync only — it does not alter their playing.

### How to run

```bash
source .venv/bin/activate
python musician.py
```

### Tempo setup — nod 4 times

Nod your head 4 times at the desired tempo before playing. The 4 nods = one bar of 4/4. On the 4th nod the MIDI clock starts and Reaper locks in.

```
Nod 1  ●○○○   recording...
Nod 2  ●●○○   recording...
Nod 3  ●●●○   recording...
Nod 4  ●●●●   → CLOCK STARTS at detected BPM
```

### Autocorrelation refinement

Once you start playing, the script reads your MIDI notes directly and runs autocorrelation analysis every 3 seconds (needs at least 8 notes). This refines the BPM to ±0.2 accuracy — the clock adjusts silently, you feel nothing change.

| Phase | Method | Accuracy |
|-------|--------|----------|
| Before playing | Head nods | ±3–5 BPM |
| After 8+ notes | Autocorrelation | ±0.2 BPM |

### HUD indicators

- `BPM: 120` — current tempo (green = running, yellow = waiting)
- `●●●●` — nod history dots, fill up as you nod
- `MIDI IN: [port]` — confirms keyboard is detected
- `REFINE: NOD ONLY` → `REFINE: AUTO` — lights up cyan when autocorr has fired

### Reaper setup

1. Transport → Synchronization → set slave to external MIDI clock
2. Select `MusicianClock` as the clock source
3. Time signature: 4/4

### Keys

- `r` — reset BPM history to 120
- `s` — stop / start clock
- `q` — quit

## Track 2 — Dancer (`gesture_midi.py`)

No music knowledge needed. Hands control notes, face and body control the style of what MRT2 generates.

### How to run

```bash
source .venv/bin/activate
python gesture_midi.py
```

On first run, three models download automatically (~45 MB total).

Then in MRT2 Jam → **MIDI INPUT** → select **GestureInstrument**.

### Gestures — notes (MIDI channel 1)

| Hand | Gesture | Effect |
|------|---------|--------|
| Left hand | Wrist up/down | Pitch (C3 at bottom → C6 at top) |
| Right hand | Index finger extended | Note ON |
| Right hand | Fist | Note OFF |
| Right hand | Thumb–index distance | Velocity (wider = louder) |

### Expression — MRT2 parameters (MIDI channel 2)

| Signal | CC | Controls |
|--------|----|---------|
| Mouth open | CC 20 | Style intensity |
| Eyebrows raised | CC 21 | Chaos |
| Smile | CC 22 | Brightness |
| Body movement energy | CC 23 | Density |
| Left/right lean | CC 24 | Pitch range |
| Arms spread | CC 25 | Overall strength |

The bottom-left panel shows 6 live bars — one per parameter.

## Two-camera setup

One machine, two cameras:

```
Built-in camera (index 0) → dancer (gesture_midi.py)
iPhone via Continuity Camera (index 1) → musician (musician.py)
```

To verify cameras are detected:

```bash
python3 -c "
import cv2
for i in range(4):
    cap = cv2.VideoCapture(i)
    if cap.isOpened():
        print(f'Camera {i}: available')
        cap.release()
"
```

To assign a specific camera, edit the `cv2.VideoCapture(index)` line at the top of each script.

## Running both tracks together

Open two terminals:

```bash
# Terminal 1 — musician
source .venv/bin/activate && python musician.py

# Terminal 2 — dancer
source .venv/bin/activate && python gesture_midi.py
```

Start MRT2 Jam and Reaper before running the scripts so MIDI ports are visible.

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Camera won't open | System Settings → Privacy → Camera → allow Terminal, restart Terminal |
| MRT2 can't see GestureInstrument | Start `gesture_midi.py` first, then open MRT2 |
| Reaper can't see MusicianClock | Start `musician.py` first, then open Reaper |
| MIDI keyboard not detected | Run `musician.py` — it prints all available ports on startup |
| Nod detection too sensitive | Move further from camera or improve lighting |
| BPM drifts | Play steadily for a few bars — autocorr will correct it |
